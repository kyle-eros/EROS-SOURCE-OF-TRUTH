ML Algorithm Components
🎯
Bayesian Smoothing
posterior = (prior_α + data) / (prior_β + n)
Handles cold-start problem by incorporating prior beliefs. Uses 100 prior sends with 5% purchase rate to smooth estimates for new captions.
🎰
Upper Confidence Bound
UCB = μ + c × √(ln(N) / n)
Balances exploration vs exploitation. Adds uncertainty bonus to encourage testing of under-explored captions. C=2.0 for optimal balance.
🔄
Dynamic Cooldown
cooldown = base × (backoff ^ uses_today)
Prevents caption fatigue with exponential backoff. Base 6 hours increases by 1.5x per use. Max 3 uses per day.
⏰
Temporal Optimization
best_hour = argmax(hourly_rps)
Identifies optimal send times per caption. Tracks 24-hour performance patterns and day-of-week trends.
🎲
Epsilon-Greedy
if rand() < ε: explore() else: exploit()
10% random exploration ensures continuous testing. Configurable by strategy: explore (25%), balanced (10%), exploit (5%).
📈
Volume-Weighted RPS
RPS = Σ(revenue) / Σ(sends)
True weighted average instead of simple mean. Gives proper weight to high-volume captions for accurate performance.
